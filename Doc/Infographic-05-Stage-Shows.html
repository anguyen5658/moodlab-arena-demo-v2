Stage System Architecture

Core systems powering every Stage show

⚙ Elimination Engine

  • 8 players per show — 1 human + 7 AI contestants
  • Each round: worst performer or wrong answer is eliminated
  • Last player standing wins the show
  • Elimination animation + MC commentary on every out
AI Contestant Names
Blaze_420 CloudNine Puff_Queen KushMaster HazeRunner ChillWave TerpLord
Elimination Flow
8 Players → Round N → Wrong / Worst → Eliminated → Repeat → Winner!

🎤 MC Commentary System

  • Dynamic AI MC commentator drives every show
  • Lines keyed to MC_LINES config
  • Template variables: {show}, {player}, {round}
Line Categories
intro contestant_pick round_start correct wrong audience_react finale
intro "Ladies and gentlemen, welcome to {show}!"
wrong "Ohhh no! {player} is OUT of here!"

👥 Role System

  • Players choose a role when joining a Stage show
  • Break-time chat available between rounds for all participants
🎮
Contestant
Play the game directly. Answer questions, puff to compete, and try to survive elimination.
👀
Audience
Watch the show live. React with puff reactions, vote for favorites, and chat between rounds.

🔴 Simon Puffs
Memory · Pattern Repeat
1–50+ Players 3–8 min Red

Mechanics

  • Sequence grows each round (1 puff up to 10 puffs)
  • Comedy MC commentary every round
