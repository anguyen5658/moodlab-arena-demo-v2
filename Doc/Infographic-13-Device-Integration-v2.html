⏱
15-Second Safety Timeout
If PUFF_STOP is never received (e.g., Bluetooth dropout), a setTimeout(15000) automatically fires btPuffUp.current?() and sets btPuffActive = false. The timeout is cleared on each new PUFF_START to prevent double-firing.

SECTION 5

Trivia Puff-to-Answer

Vibe Check and Survival Trivia use puff duration to select answers A through D. The function getTriviaPuffAnswer(ms) maps hold time to one of four options. A live highlight updates while the player is still puffing.

const getTriviaPuffAnswer = (ms) =>
  ms < 800 ? 0 // A
  : ms < 2000 ? 1 // B
  : ms < 3500 ? 2 // C
  : 3; // D

SECTION 6

Per-Render IIFE Pattern

The BLE notification listener is registered once at connect time. To avoid stale closures, a per-render IIFE writes the current game's handler closures into refs on every render. The listener always reads from refs, never from closed-over values.

// Runs as top-level IIFE inside component body (not in useEffect)
(() => {
  const id = gameActive?.id;
  let down = null, up = null;
  if (id === "finalkick") { down = kickStartCharge; up = kickStopCharge; }
  else if (id === "balloon") { down = bpStartCharge; up = bpStopCharge; }
  // ... 33 more branches ...
  btPuffDown.current = down;
  btPuffUp.current = up;
  // Puff Events always wired regardless of active game
  btPuffEventDown.current = puffEventHoldDown;
  btPuffEventUp.current = puffEventHoldUp;
})();
